Map Component Usage Examples

This page demonstrates different ways to use the map component. The component can be embedded in any HTML page and configured via data attributes.

Example 1: Read-Only Map (by name)

A simple read-only map viewer. Users can view pins but cannot add, edit, or delete them.

<div data-map-component 
     data-map-name="ansalon"
     data-editable="false"
     data-height="400px">
</div>

Example 2: Editable Map with Title

An editable map that allows users to add, edit, and delete pins. Shows the map title above the viewer.

<div data-map-component 
     data-map-name="ansalon"
     data-editable="true"
     data-show-title="true"
     data-height="500px">
</div>

Example 3: Loading Map by ID

You can also load maps using their numeric ID instead of name.

<div data-map-component 
     data-map-id="1"
     data-editable="false"
     data-height="400px">
</div>

Example 4: Compact View

A smaller, compact version for embedding in sidebars or smaller sections.

<div data-map-component 
     data-map-name="ansalon"
     data-editable="false"
     data-height="300px">
</div>

Configuration Options

Data Attributes:

  • data-map-component - Required. Identifies this as a map component.
  • data-map-name - Map name to load (case-insensitive).
  • data-map-id - Alternative: Load map by numeric ID.
  • data-editable - "true" or "false" (default: false). Enables add/edit/delete functionality.
  • data-height - CSS height value (default: "500px").
  • data-show-title - "true" or "false" (default: false). Shows map title above viewer.

Required Scripts:

Add these to your HTML page:

<!-- In <head> -->
<link rel="stylesheet" href="https://unpkg.com/leaflet@1.9.4/dist/leaflet.css" />

<!-- Before closing </body> -->
<script src="https://unpkg.com/leaflet@1.9.4/dist/leaflet.js"></script>
<script type="module" src="./assets/js/map-component.js"></script>
